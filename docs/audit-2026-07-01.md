## 1. VERİ & ZAMAN MİMARİSİ (Soru #2: hangi zaman aralığındaki veriyi kullanıyoruz?)

| Kline — 15m | **ANA KARAR** | 120 × 15dk ≈ **30 saat** | her tarama |

## 2. SİNYAL GEÇERLİLİK SÜRESİ / TTL (Soru #3: bir karar ne kadar geçerli?)

**Mevcut durum:** Karar 15m kapanış mantığı + 5m tetik üzerine kurulu, tarama her 5 dakikada (`loop_minutes: 5`). Aday üretildiği tikte **saniyeler içinde** tüketiliyor; **açık bir geçerlilik süresi (TTL) tanımlı DEĞİL**. Limit girişte emrin masada bekleme süresi yalnız `maker_timeout_s: 10s`. Bayat-sinyal koruması yok.

**Teorik geçerlilik:** Bir setup, üretildiği **5m mumun ömrü** boyunca (en fazla bir sonraki taramaya, ~5dk) anlamlıdır. 15m karar bağlamı 1-2 mum (~15-30dk) dayanır ama 5m tetik/giriş-zonu çok daha çabuk bayatlar. Tape onayı yalnız **o anki 22 saniye** için geçerlidir — order-flow saniyeler içinde döner.

**Önerilen açık TTL politikası:**

| Parametre | Öneri | Gerekçe |
|---|---|---|
| `signal_ttl_s` | **150–180 sn** | 5m tarama kadansı içinde; bu süre sonunda yeni tarama zaten yeniden değerlendirir |
| Bayat-fiyat guard'ı | mark, hedef girişten **> 0.33R** uzaklaştıysa → İPTAL | Kaçan setup'ı kovalama; kötü fill'i önler |
| Tape tazeliği | tape skoru giriş anından **> 60sn** eskiyse → yeniden tape | order-flow saniyelik döner |
| Limit resting (Bölüm 6) | `min(signal_ttl_s, maker_timeout_s)` | emir bayat sinyalle masada kalmasın |

```python
# [INFO] Sinyal hâlâ geçerli mi? Üretildikten signal_ttl_s geçtiyse veya fiyat hedef girişten 0.33R'den fazla
# [INFO] uzaklaştıysa (kaçan setup) False döner -> giriş iptal edilir, kötü/bayat dolum önlenir.
def signal_still_valid(cand, mark, now_ts, ttl_s=180, max_drift_r=0.33):
    if now_ts - getattr(cand, "created_ts", now_ts) > ttl_s:
        return False, "sinyal bayat (TTL aşıldı)"
    r_unit = abs(cand.entry - cand.stop)
    if r_unit > 0 and abs(mark - cand.entry) / r_unit > max_drift_r:
        return False, "fiyat girişten uzaklaştı (kaçan setup)"
    return True, ""
```

**Etki:** TTL + drift guard, hem market hem limit girişte "kaçmış fiyatı kovalama" hatasını keser; özellikle Bölüm 6'daki limit stratejisinin temelidir (emri ne kadar bekleteceğimizi bu belirler).

## 3. EN KRİTİK 5 BULGU (Risk Raporu)

### 🔴 #1 — Girişte SL konamazsa çıplak + kayıtsız pozisyon
`executor.enter()` → `_acquire_fill()` market dolumu yapıyor, **sonra** `place_stop_market()` çağrılıyor. Bu Algo-emir çağrısı exception fırlatırsa (gerçekçi: hızlı piyasada fiyat tetiği geçmiş → Binance **-2021 "Order would immediately trigger"**; ya da -1111/-4120), exception `enter()`'dan çıkar; `loop.run_forever`'ın genel `except`'i yalnız loglar.

Sonuç: borsada **SL'siz açık pozisyon** + DB'de **kayıt yok** → trailing göremez, reconcile göremez. Çift-giriş guard'ı yeni girişi engeller ama **mevcut çıplak pozisyonu korumaz**. En çok stop'a ihtiyaç duyulan anda (sert hareket) tetiklenir. **EN ACİL DÜZELTME.**

### 🔴 #2 — İşlem başına %25 risk + kill-switch kapalı + 3 eşzamanlı
`config.yaml`: `risk_per_trade_pct: 25.0`, `max_concurrent: 3`, `daily_max_loss_pct: 0`, tek backstop `max_consecutive_losses: 3`. Tam-güven işlemde SL = kasanın **%25'i**. 3 eşzamanlı = tek seferde **~%75 maruziyet**. 3 ardışık kayıp = halt'tan önce **~%75 drawdown**. Senin bu turdaki "%1-2 kuralı" talebinle taban tabana zıt. `dynamic_sizing.min_frac=0.30` tabanı küçültür ama **tavan %25 kalıyor**.

### 🟠 #3 — Rate-limit (429) / saat-kayması (-1021) işlenmiyor
`binance._signed/_public` non-200'de direkt fırlatıyor; **Retry-After/backoff/weight-takibi yok**. `sync_time()` yalnız açılışta bir kez. Uzun uptime'da NTP drift → `-1021` → tüm imzalı çağrılar kırılır, otomatik re-sync yok. 429 burst'ünde `poll_tick.reconcile` sessizce no-op → çıkış tespiti o döngüde yapılmaz (borsa SL/TP korur ama bot körleşir).

### 🟠 #4 — PnL/R yaklaşık; funding tamamen yok sayılıyor (ölçüm zehirlenmesi)
`executor.flatten()`: `pnl = (exit-entry)*qty*direction`, fee sabit `0.0005`, **funding hiç hesaba katılmıyor**. `_live_exit_price` çıkışı `user_trades[-1]`'den alıyor (yanlış-atıf riski). Learner + analyst + brain + tüm gün-sonu raporları bu yaklaşık R'ler üzerine kurulu → her "kendini iyileştirme" kararı **gürültülü ground-truth** üzerinde optimize ediliyor.

### 🟡 #5 — Trailing cancel→place çıplak penceresi + idempotency yok
`trailing._replace_sl`: önce eski SL iptal, sonra yeni — arada ~100-300ms stop'suz. Emirlerde `newClientOrderId` yok → retry eklenince çift-dolum riski.

## 6. LİMİT-EMİR STRATEJİSİ — FEE OPTİMİZASYONU (Soru #4)

### 6.1 Mevcut durum (çelişki çözüldü)
Config'te `entry.maker_enabled: true` ve maker-first GTX **çalışıyor**, ama son 12 girişin ~7'si **taker**'a düşmüş. Kalıp: **SHORT/momentum → taker**, LONG/pullback → maker. Neden: GTX limit en-iyi-bid/ask'e konuyor; fiyat işlemin yönüne kaçınca (SHORT'ta düşerken ask aşağı kaçar) resting emir dolmaz → 10s timeout → taker top-up. Yani momentum girişlerinde maker mantığı fiilen işe yaramıyor → gereksiz taker fee.

**Fee farkı (USD-M Futures, yaklaşık):** taker ~%0.05, maker ~%0.02 → giriş başına ~%0.03 tasarruf. ~67 işlemde mütevazı ($5-9) ama **birikimli ve sürekli**; ayrıca daha iyi giriş fiyatı (slippage azalır).

### 6.2 Seçilen yaklaşım: DENGE (akıllı limit + garanti dolum)
Akıllı limit dene; TTL (Bölüm 2) sonunda **taker fallback** ile işlemi **asla kaçırma**. Yani fee tasarrufu fırsatçı, dolum garantili.

### 6.3 Limit fiyatını belirleyen yeni gösterge/metrikler
Emri "en-iyi-bid'e koy" yerine **setup mantığına göre** yerleştir:

| Setup tipi | Limit yerleşimi | Dayanak metrik | Beklenen davranış |
|---|---|---|---|
| Pullback / RANGE / mean-reversion (LONG-uptrend-pullback) | EMA21 retest / setup giriş-zonu seviyesine | `setup_levels` zonu, EMA21 bandı | Fiyat zaten geri çekildi → yüksek maker dolum şansı, sıfır taker |
| Momentum / FRESH (fiyat kaçıyor) | en-iyi-bid/ask + küçük ATR ofseti, **kısa** TTL | `book_ticker`, ATR | Kısa bekle; dolmazsa hızlı taker (kovalama yok) |
| Likidite-mıknatısı yakın | mıknatıs/retest seviyesine | `liq_pull` / liqmap | Fiyatın geri geleceği seviyeye limit |
| Güçlü defter-vakumu yönde | duvarın hemen önüne | `book_asym` + depth | Duvar emilmeden iyi fiyattan maker |
| Funding lehte | daha uzun resting (TTL üst sınırına yakın) | funding işareti | Maker şansını artır |

### 6.4 Akış (taslak)
```python
# [INFO] Akıllı limit giriş fiyatını setup tipine göre hesaplar: pullback'te retest/EMA21 zonuna, momentumda
# [INFO] en-iyi-fiyat + küçük ATR ofsetine koyar. Mıknatıs/defter-duvarı varsa onu önceler. round_price ile ticke yuvarlar.
def smart_limit_price(binance, cand, mark, atr):
    bt = binance.book_ticker(cand.symbol)
    best = float(bt["bidPrice"]) if cand.side == "LONG" else float(bt["askPrice"])
    zone = getattr(cand, "entry_zone", None)        # setup_levels'tan pullback/retest seviyesi
    magnet = getattr(cand, "liq_pull_level", None)   # liqmap mıknatıs seviyesi
    if cand.status in ("PULLBACK-MOM", "IZLE-pullback") and zone:
        px = zone                                    # mean-reversion: retest zonuna (yüksek maker şansı)
    elif magnet is not None:
        px = magnet                                  # mıknatıs seviyesine
    else:
        off = 0.10 * atr                             # momentum: en-iyi fiyat + küçük ATR ofseti
        px = best - off if cand.side == "LONG" else best + off
    return binance.round_price(cand.symbol, px)
```

```python
# [INFO] DENGE dolum motoru: akıllı limit fiyatına GTX post-only koyar, signal-TTL kadar bekler; ilk dolumda SL'yi
# [INFO] HEMEN koyar (çıplak pencere yok). TTL sonunda kalanı reduceOnly market ile tamamlar -> işlem ASLA kaçmaz.
def acquire_fill_balanced(cfg, binance, cand, mark, sl_price, qty, atr, ttl_s):
    entry_side, exit_side = _sides(cand.side)
    px = smart_limit_price(binance, cand, mark, atr)
    lo = binance.place_limit(cand.symbol, entry_side, qty, px, tif="GTX",
                             client_id=f"cfs-{cand.symbol}-entry")   # idempotent (çift-dolum yok)
    filled, avgpx, sl_order = 0.0, mark, None
    deadline = _now() + ttl_s
    while _now() < deadline:
        _sleep(cfg.get("entry", {}).get("maker_poll_s", 1.0))
        od = binance.get_order(cand.symbol, lo.get("orderId"))
        ex = float(od.get("executedQty", 0) or 0)
        if ex > filled:
            filled = ex
            if sl_order is None:                       # İLK dolum -> SL HEMEN
                sl_order = binance.place_stop_market(cand.symbol, exit_side, sl_price, close_position=True)
        if str(od.get("status", "")).upper() == "FILLED":
            return filled, avgpx, lo, sl_order, "maker"
    binance.cancel_order(cand.symbol, lo.get("orderId"))           # TTL doldu
    remainder = binance.round_qty(cand.symbol, qty - filled)
    if remainder > 0:                                              # garanti dolum: kalanı market
        binance.place_market(cand.symbol, entry_side, remainder)
        if sl_order is None:
            sl_order = binance.place_stop_market(cand.symbol, exit_side, sl_price, close_position=True)
        return qty, mark, lo, sl_order, ("maker+taker" if filled > 0 else "taker")
    return filled, avgpx, lo, sl_order, "maker"
```

**Not:** `entry_zone` ve `liq_pull_level` motorda zaten kısmen var (`setup_levels`, liqmap); bu alanların adaya taşınması küçük bir köprü gerektirir (engine'e DOKUNMADAN, cfs-trader tarafında okunur).

### 8.1 #1 — SL-garantili giriş (acil)
```python
# [INFO] Girişi SL-garantili yapar: SL borsaya yerleşemezse pozisyonu acil reduceOnly-market ile kapatır
# [INFO] (çıplak bırakmaz) ve durumu loglar. Dolum her hâlükârda DB'ye yazılır ki orphan reconcile tarafından görülebilsin.
def enter(cfg, binance, store, cand, sizing, mark, day):
    sym = cand.symbol
    entry_side, exit_side = _sides(cand.side)
    fill_qty, fill_px, entry_order, sl_order, mode = _acquire_fill(...)
    try:
        if sl_order is None:
            sl_order = _place_sl_resilient(binance, sym, exit_side, sl_price)  # -2021 -> geri-çekik tetik / reduceOnly
    except Exception as e:
        binance.place_market(sym, exit_side, fill_qty, reduce_only=True)        # ÇIPLAK BIRAKMA: acil kapat
        store.log_decision(sym, cand.side, "ABORT_NO_SL", repr(e))
        if cfg.notifier: cfg.notifier.send(f"⛔ {sym}: SL konamadı, pozisyon acil kapatıldı")
        return None
    tid = store.open_trade(...)   # SL kesinleştikten sonra kayıt
    return tid
```

### 8.2 #3 — Dayanıklı imzalı istek (429 / -1021 / backoff / idempotency)
```python
# [INFO] Tüm REST çağrılarını dayanıklı yapar: 429/418'de Retry-After bekler, -1021 saat-kaymasında zamanı yeniden
# [INFO] senkronize edip tekrar dener, ağ hatasında üstel backoff uygular. Emir POST'larına clientOrderId ekleyince retry idempotenttir.
def _request(self, method, path, params, signed, retries=3):
    for i in range(retries + 1):
        try:
            r = self._s.request(method, self._url(path, params, signed), timeout=15)
            if r.status_code in (429, 418):
                time.sleep(float(r.headers.get("Retry-After", 2 ** i))); continue
            if r.status_code != 200:
                if "-1021" in r.text:
                    self.sync_time(); continue          # saat kaymasını düzelt, tekrar dene
                raise BinanceError(f"{method} {path} {r.status_code}: {r.text[:200]}")
            return r.json()
        except requests.exceptions.RequestException:
            if i == retries: raise
            time.sleep(2 ** i)                            # üstel backoff
```

### 8.4 #4 ölçüm — ground-truth PnL (funding dahil)
```python
# [INFO] Gerçek net PnL'i borsadan çeker: realizedPnl + commission + funding (income endpoint). Funding artık dahil;
# [INFO] learner/analyst bundan sonra GERÇEK net R üzerinde öğrenir (tahmini fee/funding-yok-sayma zehiri temizlenir).
def realized_pnl(binance, sym, since_ts):
    inc = binance.income(sym, since=since_ts)   # GET /fapi/v1/income
    g = lambda t: sum(float(x["income"]) for x in inc if x["incomeType"] == t)
    return g("REALIZED_PNL") + g("COMMISSION") + g("FUNDING_FEE")
```
